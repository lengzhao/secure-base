# Phase 0 产品决议

> 日期：2026-06-14（通道修订：同日）  
> 状态：**已决** — 指导 [Phase 0 Agent 产品](../design/phase-0-agent-product.md) 落地  
> 依据：产品方向讨论；OpenClaw [飞书](https://docs.openclaw.ai/channels/feishu) / [微信](https://docs.openclaw.ai/channels/wechat) 通道可行性确认

## 1. MVP 通道：飞书 + 微信

**决议**

Phase 0 支持 **飞书** 与 **微信** 两个 IM 通道；用户 onboarding 时 **二选一**，均通过 **App 扫码** 完成 Bot / Agent 登录。

| 通道 | 技术栈 | 绑定方式 |
|------|--------|----------|
| **飞书** | OpenClaw 内置 [Feishu 通道](https://docs.openclaw.ai/channels/feishu)，WebSocket | 飞书 App 扫码创建 / 关联 Bot |
| **微信** | 腾讯官方 [`@tencent-weixin/openclaw-weixin`](https://github.com/Tencent/openclaw-weixin) 插件，ilink | 微信 App 扫码授权（`openclaw channels login --channel openclaw-weixin`） |

| 维度 | 说明 |
|------|------|
| 套餐 | 基础 / 标准档均含 **1 个通道**；注册时选定，Phase 0.2 再评估双通道同绑 |
| 隔离 | 每用户独立 Gateway 实例 + 独立通道凭证 |
| 平台基线 | OpenClaw ≥ **2026.3.22**（微信插件）；飞书通道取官方文档较高版本要求 |
| 暂不做 | Telegram、Discord、QQ、企业微信（二期评估） |

**理由**：国内用户覆盖飞书（办公）与微信（日常）；两通道均支持扫码 onboarding，符合「5 分钟可用」；微信采用 **腾讯官方插件**，非第三方个人号桥接，降低封号风险；抖音演示可同时展示两条绑定路径。

**微信说明**：不使用网页版微信桥接或实验性个人号方案；群聊能力按插件迭代，MVP 以私聊为主。

## 2. 法律主体：公司

**决议**

Phase 0 以 **有限责任公司** 作为唯一对外运营与签约主体。

| 维度 | 说明 |
|------|------|
| 职责 | SaaS 订阅、开票、雇佣、IM 开放平台 / 插件合规主体 |
| 集体池 | 可分配产出 **≥ 51%** 转入公司名下 **安心基座专项银行账户** |
| 公司留存 | **≤ 49%** 用于产品、安全、客服、税务 |
| 公开 | 月度披露入池金额与专项账户累计余额 |
| 与 Phase 3 | Phase 0 为公司内部分配；正式 51/49 股权结构留 Phase 3 |

**待办**：公司注册、专项账户开立、股东决议或章程附件写入分配比例。

**理由**：订阅 SaaS 在国内以公司主体最清晰；便于对接支付、发票与开放平台；专项账户可在不改变公司形式下执行 51% 承诺。

## 3. 种子成员来源：抖音

**决议**

种子成员（50–100 人）从 **抖音** 内容转化，形成 **线上认同圈**；**不要求**与 Phase 1 线下试点同一地理范围。

| 维度 | 说明 |
|------|------|
| 获客 | 抖音短视频 / 直播 → 备案落地页 → 注册 → **选飞书或微信** → 扫码绑定 |
| 种子权益 | 基础档 6 个月免费或 5 折；换 baseline 问卷与可选访谈 |
| 追踪 | 邀请码 / UTM，`source=douyin` |
| Phase 1 | 线下试点城市 **另行选定**；抖音种子为产品与叙事验证，非 Phase 1 地理社区 |

**理由**：冷启动不依赖 500 人同城社区；「AI 养人」叙事适合短视频；**飞书 / 微信双通道绑定**均可作演示内容，微信路径对抖音受众更友好。

**内容红线**：不承诺理财回报；强调工具订阅 + 可分配产出 51% 公开回流集体池。
